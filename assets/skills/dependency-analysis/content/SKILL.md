# Dependency analysis

1. Inventory direct framework, SDK, runtime, workload, and package constraints.
2. Locate central version declarations, lock files, build props/targets, containers, deployment manifests, and CI matrices.
3. Identify compatibility boundaries and transitive risks using authoritative release and compatibility documentation.
4. Produce an ordered upgrade plan with rollback points.
5. Validate restore, build, tests, analyzers, packaging, and the repository's deployment checks.

Never claim compatibility from a successful restore alone.
